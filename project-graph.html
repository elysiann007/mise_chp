⚠️ Structure Issues Found
Problems in the current codebase that need to be cleaned up before scaling
🗑️
Backend: Boilerplate files never removed MEDIUM
app.controller.ts, app.service.ts, app.controller.spec.ts are the NestJS "hello world" scaffold. They add noise and are not wired to anything meaningful. Should be deleted.
👻
Backend: 6 ghost modules — exist on disk, not in AppModule HIGH
bills/, fiscal/, payments/, restaurants/, menu/, notifications/ — folders exist with subdirectory structure but contain no implementation files and are NOT imported in AppModule. Either implement them or remove the folders. Right now they create confusion about what is and isn't built.
📂
Backend: Empty subdirectory pollution inside auth/ LOW
auth/decorators/ and auth/guards/ are empty — the actual implementations live in common/decorators/ and common/guards/ (correct place). Delete the empty auth subdirs to avoid confusion.
⚛️
Frontend: Admin pages have both flat files AND sub-directories HIGH
e.g. pages/admin/Dashboard.tsx exists AND pages/admin/Dashboard/ folder exists (empty). Same for MenuManager, TableManager. The empty subdirs suggest a planned refactor to co-located components that was never completed. Remove empty dirs or complete the refactor — pick one pattern.
🔀
Frontend: Inconsistent staff page location MEDIUM
Kitchen is at pages/staff/Kitchen/Kitchen.tsx but there's also a reference to pages/kitchen/KitchenDisplay/. One is real, one is an empty ghost. Clean up the pages/kitchen/ ghost dir.
📦
Frontend: shadcn/ui dependency without shadcn components LOW
lib/utils.ts exports the cn() helper (clsx + tailwind-merge) — this is the shadcn pattern. But no shadcn components are installed. Either install shadcn components to leverage this properly, or it just means Tailwind class merging is already available.
🌍
Frontend: i18next installed but i18n/ folder is empty MEDIUM
Both i18next and react-i18next are in package.json, but src/i18n/ has no translation files and no initialisation. Either wire it up (important for tr-TR support) or remove the dependency.
🖼️
Frontend: Vite/React boilerplate assets never cleaned LOW
assets/react.svg, assets/vite.svg, App.css (Vite starter CSS), and assets/hero.png are leftovers from project initialization. Should be deleted.
📁
Frontend: 5 empty component/config folders LOW
components/common/, components/forms/, components/layout/, components/shared/, styles/, config/, constants/ — all empty. These are placeholders. Remove until actually needed.
💰
MVP Gap: Bills & Payments not started NEXT UP
No BillsModule, no PaymentsModule — iyzico 3DS Hosted Form integration is the critical next feature. The folder skeletons exist but have zero implementation.
📄
MVP Gap: Fiscal documents (Paraşüt e-Arşiv) not started NEXT UP
No Paraşüt integration, no GİB XML builder, no FiscalModule. Folder skeleton exists in fiscal/providers/parasut/ but empty.

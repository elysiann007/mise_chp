# CafeHookahPub — 4-Agent Code Review (2026-05-18)

## Source
Four parallel review agents: Architecture, UX/Features, SEO/Deployment, i18n completeness.

## Dead Code Identified

- `src/services/api/client.ts` — unused axios HTTP client, zero call sites, MISE SaaS leftover
- `src/types/entity.types.ts` — full MISE entity type library (Restaurant, Table, Order, etc.), zero references
- Unused packages: socket.io-client, @tanstack/react-query, react-hook-form, zod, qrcode, date-fns, axios

## Bugs Found

- `HookahBuilder.updatePct()` has no clamp — slider can push total past 100%; fix: `Math.max(5, Math.min(100, v))`
- Disabled aroma buttons still fire Framer Motion whileHover/whileTap — visual glitch on interaction
- Animation duration inconsistency: fadeUp uses 0.7s, fadeIn uses 0.6s

## Missing Features

- No Contact / Reservation page — phone number is placeholder (+90000000000)
- No Gallery / photo section — 120-seat venue with zero images
- No Events / promotions page
- No 404 page — wildcard route silently redirects to Home
- "Order This Blend" button (HookahBuilder) has no click handler
- HookahBuilder blend not persisted to localStorage — lost on refresh
- No social media links in Footer

## SEO & Deployment Gaps

- index.html missing: meta description, og:title, og:description, og:image, Twitter cards, canonical URL
- .env hardcodes VITE_API_URL=localhost:3000 — Vercel deploy blocker
- No vercel.json — SPA routing not configured for Vercel
- No robots.txt, no sitemap.xml
- No Schema.org local business JSON-LD

## i18n Issues

- Hardcoded English strings bypassing i18n:
  - Navbar.tsx aria-label for hamburger button
  - HookahBuilder.tsx aria-label for aroma remove button
  - Footer.tsx operating hour times (5 PM – 2 AM, 4 PM – 4 AM, 5 PM – 1 AM)
- RTL gradient issue: bg-gradient-to-r/l in Home.tsx don't flip for Arabic — need rtl: variants

## What is Working Well

- All 6 locale files (en, tr, es, it, ru, ar) have identical key structures
- Language detection and localStorage persistence is robust
- RTL dir toggle working correctly for Arabic
- TypeScript strict mode enabled, clean build
- Framer Motion animation system is well-structured and consistent
- Mobile layout is solid — no broken flows
- Google Fonts loaded with display=swap and preconnect

## Refactors Recommended

- Extract AROMAS data to src/constants/aromas.ts (HookahBuilder is 433 lines)
- Extract MODELS to src/constants/hookahModels.ts
- Create src/constants/venue.ts for phone, address, mapUrl, email
- Move aroma percentage allocation logic to src/utils/calculateAromaAllocation.ts
